Introduction

À propos du projet

Ce projet a pour objectif de construire un pipeline complet d'analyse des avis clients en ligne, depuis la collecte automatisée (webscraping) jusqu'à l'analyse de sentiments et la visualisation. Il cible principalement les restaurants de Dakar, Sénégal via TripAdvisor et Google Maps, avec une extension aux produits e-commerce via Jumia et Amazon.fr.

Le projet est développé en équipe et réparti sur deux dépôts GitHub complémentaires : l'un dédié à l'analyse NLP et la base de données, l'autre au webscraping et à l'application mobile Flutter.

🎯

Objectif principal

Collecter et analyser automatiquement les avis clients pour produire des insights sur la réputation des établissements.

🤝

Projet d'équipe

Chaque membre scrape une plateforme différente. Les données sont centralisées dans un format JSON unifié.

🧠

NLP intégré

Analyse de sentiments, extraction de mots-clés et classification des avis via un pipeline NLP Python.

📱

Application Flutter

Une application mobile Dart/Flutter (mainbase.dart) pour consommer et visualiser les données.

Scraping

Spider TripAdvisor

Le spider principal utilise Selenium avec Chrome en mode furtif pour collecter les avis de restaurants sur TripAdvisor. Il scrape automatiquement toutes les pages d'avis pour chaque restaurant de la liste Dakar.

Étape 1

Chrome en mode furtif

Chrome lancé avec suppression des flags d'automatisation, user-agent Chrome 144 réaliste, désactivation de navigator.webdriver via JavaScript et profil temporaire dans C:\Temp\chrome_selenium.

Étape 2

Collecte des URLs (35 restaurants)

Ouverture de la page liste TripAdvisor Dakar, acceptation automatique des cookies RGPD, extraction de tous les liens Restaurant_Review. URLs normalisées (suppression du fragment #REVIEWS) avant déduplication.

Étape 3

Extraction par restaurant

Pour chaque restaurant : nom via premier H1 non vide, cartes via div[data-automation='reviewCard'], note via système multi-fallback, texte complet (clic sur "Plus" si tronqué), date et auteur.

Étape 4

Pagination automatique

10 avis par page. URL suivante construite par injection de l'offset (-or10-, -or20-...). Trois conditions d'arrêt : page vide, total atteint, moins de 10 avis. Limite : 50 pages max.

Débogage

Problèmes rencontrés & solutions

Problème #1

Spider Scrapy non trouvé

TripAdvisor charge les avis en JavaScript dynamique — Scrapy seul ne peut pas les voir.

Solution

Migration vers Selenium pur

Chrome complet via Selenium exécute le JavaScript comme un vrai navigateur.

Problème #2

Notes toutes à 0.0⭐

Sélecteur CSS svg.evwcZ ne matchait plus — classes CSS dynamiques qui changent à chaque déploiement.

Solution

4 stratégies de fallback

Sélecteur d'attribut svg[aria-label] + 3 autres méthodes de récupération de la note.

Problème #3

URLs dupliquées

Liens avec et sans fragment #REVIEWS causaient le double scraping des mêmes restaurants.

Solution

Normalisation des URLs

href.split("#")[0] avant ajout au set de déduplication.

Problème #4

SQLite unable to open database

Lancement depuis un sous-dossier — SQLite cherchait database/reviews.db dans le mauvais répertoire.

Solution

Fix du CWD

os.chdir(os.path.dirname(os.path.abspath(__file__))) en tête de chaque script.

Problème #5

Boucle de pagination infinie

Bouton "Page suivante" toujours détecté comme actif — sélecteurs CSS incorrects.

Solution

3 conditions d'arrêt

Page vide + total atteint + moins de 10 avis. Limite absolue : 50 pages max.

Problème #6

Blocage anti-bot TripAdvisor

Après scraping intensif, TripAdvisor a bloqué les requêtes — pages vides, 0 éléments trouvés.

Solution

Reset profil Chrome

Remove-Item -Recurse -Force "C:\Temp\chrome_selenium" + pauses aléatoires entre sessions.

Problème #7

Classes CSS du H1 changées

Après reset du profil, le sélecteur h1.biGQs._P.fiohW.eIegw ne matchait plus rien.

Solution

Sélecteur agnostique

Tous les <h1> récupérés, premier avec texte non vide sélectionné — résistant aux changements CSS.

Problème #8

Encodage corrompu JSON

Fichiers Jumia et Google Maps avec Ã© au lieu de é — confusion latin-1 / UTF-8.

Solution

Correction automatique à l'import

text.encode('latin-1').decode('utf-8') dans import_reviews.py.

Pipeline données

Consolidation des sources

Chaque membre de l'équipe scrape une plateforme différente et produit un fichier JSON. Ces fichiers sont ensuite consolidés dans la base SQLite partagée via un script d'import universel.

Étape 1

Export SQLite → JSON

Les données TripAdvisor et Amazon scrapées via Selenium sont exportées en JSON : python export_tripadvisor.py et python export_amazon.py.

Étape 2

Correction encodage

La fonction fix_encoding() dans import_reviews.py corrige automatiquement les fichiers Jumia et Google Maps avec caractères corrompus.

Étape 3

Import unifié

import_reviews.py accepte plusieurs fichiers JSON, valide la plateforme, évite les doublons (auteur + produit + date + plateforme) et insère dans SQLite.

Étape 4

Nettoyage doublons Amazon

clean_amazon_duplicates.py a supprimé 149 enregistrements avec anciens noms incorrects, ramenant de 297 à 148 avis Amazon propres.

import_reviews.py — fix_encoding()
def fix_encoding(text: str) -> str:
    """Corrige la corruption latin-1/UTF-8.
    'Ã©' → 'é'  ·  'â€™' → '''  ·  'Ã¨' → 'è'"""
    try:
        return text.encode('latin-1').decode('utf-8')

Analyse

Pipeline NLP

Le dépôt Review_analyser contient un pipeline NLP complet (nlp_pipeline.py, nlp_pipeline1.py) ainsi que des scripts d'évaluation (evaluate_nlp.py, evaluate_nlp1.py, evaluate_nlp2.py) pour mesurer la qualité des prédictions.

🔍

Détection de langue

Identification automatique de la langue de chaque avis (français, anglais, wolof...) via langdetect.

❤️

Analyse de sentiments

Classification positive / négative / neutre via modèle BERT ou CamemBERT fine-tuné sur les avis francophones.

🏷️

Extraction de mots-clés

Identification des thèmes récurrents : service, cuisine, ambiance, prix, propreté, etc.

📊

Évaluation du modèle

Scripts d'évaluation avec métriques précision / rappel / F1-score pour valider la qualité des prédictions.
